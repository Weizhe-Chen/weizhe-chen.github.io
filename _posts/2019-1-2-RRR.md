```python
def ucb1(numActions, reward):
   payoffSums = [0] * numActions
   numPlays = [1] * numActions
   ucbs = [0] * numActions
 
   # initialize empirical sums
   for t in range(numActions):
      payoffSums[t] = reward(t,t)
      yield t, payoffSums[t], ucbs
 
   t = numActions
 
   while True:
      ucbs = [payoffSums[i] / numPlays[i] + upperBound(t, numPlays[i]) for i in range(numActions)]
      action = max(range(numActions), key=lambda i: ucbs[i])
      theReward = reward(action, t)
      numPlays[action] += 1
      payoffSums[action] += theReward
 
      yield action, theReward, ucbs
      t = t + 1
```

## Multi-Armed Bandits

The multi-armed bandit scenario is simple to describe, and it boils the exploration-exploitation tradeoff down to its purest form.

Suppose you have a set of ![K](https://s0.wp.com/latex.php?latex=K&bg=ffffff&fg=36312d&s=0) actions labeled by the integers ![\left \{ 1, 2, \dots, K \right \}](https://s0.wp.com/latex.php?latex=%5Cleft+%5C%7B+1%2C+2%2C+%5Cdots%2C+K+%5Cright+%5C%7D&bg=ffffff&fg=36312d&s=0). We call these *actions* in the abstract, but in our minds they’re slot machines. We can then play a game where, in each round, we choose an action (a slot machine to play), and we observe the resulting payout. Over many rounds, we might explore the machines by trying some at random. Assuming the machines are not identical, we naturally play machines that seem to pay off well more frequently to try to maximize our total winnings.

![Exploit away, you lucky ladies.](https://jeremykun.files.wordpress.com/2013/10/slot-machines.jpg?w=300&h=199)

Exploit away, you lucky ladies.

This is the most general description of the game we could possibly give, and every bandit learning problem has these two components: actions and rewards. But in order to get to a concrete problem that we can reason about, we need to specify more details. Bandit learning is a large tree of variations and this is the point at which the field ramifies. We presently care about two of the main branches.

**How are the rewards produced?** There are many ways that the rewards could work. One nice option is to have the rewards for action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) be drawn from a fixed distribution ![D_i](https://s0.wp.com/latex.php?latex=D_i&bg=ffffff&fg=36312d&s=0) (a different reward distribution for each action), and have the draws be independent across rounds and across actions. This is called the *stochastic* setting and it’s what we’ll use in this post. Just to pique the reader’s interest, here’s the alternative: instead of having the rewards be chosen randomly, have them be *adversarial.* That is, imagine a casino owner *knows your algorithm* and your internal beliefs about which machines are best at any given time. He then fixes the payoffs of the slot machines in advance of each round to screw you up! This sounds dismal, because the casino owner could just make all the machines pay nothing every round. But actually we can design good algorithms for this case, but “good” will mean something different than absolute winnings. And so we must ask:

**How do we measure success?** In both the stochastic and the adversarial setting, we’re going to have a hard time coming up with any theorems about the performance of an algorithm if we care about how much absolute reward is produced. There’s nothing to stop the distributions from having terrible expected payouts, and nothing to stop the casino owner from intentionally giving us no payout. Indeed, the problem lies in our measurement of success. A better measurement, which we can apply to both the stochastic and adversarial settings, is the notion of *regret*. We’ll give the definition for the stochastic case, and investigate the adversarial case in a future post.

**Definition:** Given a player algorithm ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) and a set of actions ![\left \{1, 2, \dots, K \right \}](https://s0.wp.com/latex.php?latex=%5Cleft+%5C%7B1%2C+2%2C+%5Cdots%2C+K+%5Cright+%5C%7D&bg=ffffff&fg=36312d&s=0), the *cumulative regret* of ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) in rounds ![1, \dots, T](https://s0.wp.com/latex.php?latex=1%2C+%5Cdots%2C+T&bg=ffffff&fg=36312d&s=0) is the difference between the expected reward of the best action (the action with the highest expected payout) and the expected reward of ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) for the first ![T](https://s0.wp.com/latex.php?latex=T&bg=ffffff&fg=36312d&s=0) rounds.

We’ll add some more notation shortly to rephrase this definition in symbols, but the idea is clear: we’re competing against the best action. Had we known it ahead of time, we would have just played it every single round. Our notion of success is not in how well we do absolutely, but in how well we do *relative to what is feasible*.

## Notation

Let’s go ahead and draw up some notation. As before the actions are labeled by integers ![\left \{ 1, \dots, K \right \}](https://s0.wp.com/latex.php?latex=%5Cleft+%5C%7B+1%2C+%5Cdots%2C+K+%5Cright+%5C%7D&bg=ffffff&fg=36312d&s=0). The *reward* of action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) is a ![[0,1]](https://s0.wp.com/latex.php?latex=%5B0%2C1%5D&bg=ffffff&fg=36312d&s=0)-valued random variable ![X_i](https://s0.wp.com/latex.php?latex=X_i&bg=ffffff&fg=36312d&s=0)distributed according to an unknown distribution and possessing an unknown expected value ![\mu_i](https://s0.wp.com/latex.php?latex=%5Cmu_i&bg=ffffff&fg=36312d&s=0). The game progresses in rounds ![t = 1, 2, \dots](https://s0.wp.com/latex.php?latex=t+%3D+1%2C+2%2C+%5Cdots&bg=ffffff&fg=36312d&s=0) so that in each round we have different random variables ![X_{i,t}](https://s0.wp.com/latex.php?latex=X_%7Bi%2Ct%7D&bg=ffffff&fg=36312d&s=0) for the reward of action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) in round ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) (in particular, ![X_{i,t}](https://s0.wp.com/latex.php?latex=X_%7Bi%2Ct%7D&bg=ffffff&fg=36312d&s=0) and ![X_{i,s}](https://s0.wp.com/latex.php?latex=X_%7Bi%2Cs%7D&bg=ffffff&fg=36312d&s=0) are identically distributed). The ![X_{i,t}](https://s0.wp.com/latex.php?latex=X_%7Bi%2Ct%7D&bg=ffffff&fg=36312d&s=0) are independent as both ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0)and ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) vary, although when ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) varies the distribution changes.

So if we were to play action 2 over and over for ![T](https://s0.wp.com/latex.php?latex=T&bg=ffffff&fg=36312d&s=0) rounds, then the total payoff would be the random variable ![G_2(T) = \sum_{t=1}^T X_{2,t}](https://s0.wp.com/latex.php?latex=G_2%28T%29+%3D+%5Csum_%7Bt%3D1%7D%5ET+X_%7B2%2Ct%7D&bg=ffffff&fg=36312d&s=0). But by independence across rounds and the linearity of expectation, the expected payoff is just ![\mu_2 T](https://s0.wp.com/latex.php?latex=%5Cmu_2+T&bg=ffffff&fg=36312d&s=0). So we can describe the *best action* as the action with the highest expected payoff. Define

![\displaystyle \mu^* = \max_{1 \leq i \leq K} \mu_i](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Cmu%5E%2A+%3D+%5Cmax_%7B1+%5Cleq+i+%5Cleq+K%7D+%5Cmu_i&bg=ffffff&fg=36312d&s=0)

We call the action which achieves the maximum ![i^*](https://s0.wp.com/latex.php?latex=i%5E%2A&bg=ffffff&fg=36312d&s=0).

A *policy* is a randomized algorithm ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) which picks an action in each round based on the history of chosen actions and observed rewards so far. Define ![I_t](https://s0.wp.com/latex.php?latex=I_t&bg=ffffff&fg=36312d&s=0) to be the action played by ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) in round ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) and ![P_i(n)](https://s0.wp.com/latex.php?latex=P_i%28n%29&bg=ffffff&fg=36312d&s=0) to be the number of times we’ve played action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) in rounds ![1 \leq t \leq n](https://s0.wp.com/latex.php?latex=1+%5Cleq+t+%5Cleq+n&bg=ffffff&fg=36312d&s=0). These are both random variables. Then the cumulative payoff for the algorithm ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) over the first ![T](https://s0.wp.com/latex.php?latex=T&bg=ffffff&fg=36312d&s=0) rounds, denoted ![G_A(T)](https://s0.wp.com/latex.php?latex=G_A%28T%29&bg=ffffff&fg=36312d&s=0), is just

![\displaystyle G_A(T) = \sum_{t=1}^T X_{I_t, t}](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+G_A%28T%29+%3D+%5Csum_%7Bt%3D1%7D%5ET+X_%7BI_t%2C+t%7D&bg=ffffff&fg=36312d&s=0)

and its expected value is simply

![\displaystyle \mathbb{E}(G_A(T)) = \mu_1 \mathbb{E}(P_1(T)) + \dots + \mu_K \mathbb{E}(P_K(T))](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Cmathbb%7BE%7D%28G_A%28T%29%29+%3D+%5Cmu_1+%5Cmathbb%7BE%7D%28P_1%28T%29%29+%2B+%5Cdots+%2B+%5Cmu_K+%5Cmathbb%7BE%7D%28P_K%28T%29%29&bg=ffffff&fg=36312d&s=0).

Here the expectation is taken over all random choices made by the policy and over the distributions of rewards, and indeed both of these can affect how many times a machine is played.

Now the *cumulative regret* of a policy ![A](https://s0.wp.com/latex.php?latex=A&bg=ffffff&fg=36312d&s=0) after the first ![T](https://s0.wp.com/latex.php?latex=T&bg=ffffff&fg=36312d&s=0) steps, denoted ![R_A(T)](https://s0.wp.com/latex.php?latex=R_A%28T%29&bg=ffffff&fg=36312d&s=0) can be written as

![\displaystyle R_A(T) = G_{i^*}(T) - G_A(T)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+R_A%28T%29+%3D+G_%7Bi%5E%2A%7D%28T%29+-+G_A%28T%29&bg=ffffff&fg=36312d&s=0)

And the goal of the policy designer for this bandit problem is to minimize the expected cumulative regret, which by linearity of expectation is

![\mathbb{E}(R_A(T)) = \mu^*T - \mathbb{E}(G_A(T))](https://s0.wp.com/latex.php?latex=%5Cmathbb%7BE%7D%28R_A%28T%29%29+%3D+%5Cmu%5E%2AT+-+%5Cmathbb%7BE%7D%28G_A%28T%29%29&bg=ffffff&fg=36312d&s=0).

Before we continue, we should note that there are theorems concerning *lower bounds*for expected cumulative regret. Specifically, for this problem it is known that no algorithm can guarantee an expected cumulative regret better than ![\Omega(\sqrt{KT})](https://s0.wp.com/latex.php?latex=%5COmega%28%5Csqrt%7BKT%7D%29&bg=ffffff&fg=36312d&s=0). It is also known that there are algorithms that guarantee no worse than ![O(\sqrt{KT})](https://s0.wp.com/latex.php?latex=O%28%5Csqrt%7BKT%7D%29&bg=ffffff&fg=36312d&s=0)expected regret. The algorithm we’ll see in the next section, however, only guarantees ![O(\sqrt{KT \log T})](https://s0.wp.com/latex.php?latex=O%28%5Csqrt%7BKT+%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0). We present it on this blog because of its simplicity and ubiquity in the field.

## The UCB1 Algorithm

The policy we examine is called UCB1, and it can be summed up by the principle of **optimism in the face of uncertainty**. That is, despite our lack of knowledge in what actions are best we will construct an optimistic guess as to how good the expected payoff of each action is, and pick the action with the highest guess. If our guess is wrong, then our optimistic guess will quickly decrease and we’ll be compelled to switch to a different action. But if we pick well, we’ll be able to exploit that action and incur little regret. In this way we balance exploration and exploitation.

The formalism is a bit more detailed than this, because we’ll need to ensure that we don’t rule out good actions that fare poorly early on. Our “optimism” comes in the form of an *upper confidence bound* (hence the acronym UCB). Specifically, we want to know with high probability that the true expected payoff of an action ![\mu_i](https://s0.wp.com/latex.php?latex=%5Cmu_i&bg=ffffff&fg=36312d&s=0) is less than our prescribed upper bound. One general (distribution independent) way to do that is to use the [Chernoff-Hoeffding inequality](https://jeremykun.com/2013/04/15/probabilistic-bounds-a-primer/).

As a reminder, suppose ![Y_1, \dots, Y_n](https://s0.wp.com/latex.php?latex=Y_1%2C+%5Cdots%2C+Y_n&bg=ffffff&fg=36312d&s=0) are independent random variables whose values lie in ![[0,1]](https://s0.wp.com/latex.php?latex=%5B0%2C1%5D&bg=ffffff&fg=36312d&s=0) and whose expected values are ![\mu_i](https://s0.wp.com/latex.php?latex=%5Cmu_i&bg=ffffff&fg=36312d&s=0). Call ![Y = \frac{1}{n}\sum_{i}Y_i](https://s0.wp.com/latex.php?latex=Y+%3D+%5Cfrac%7B1%7D%7Bn%7D%5Csum_%7Bi%7DY_i&bg=ffffff&fg=36312d&s=0) and ![\mu = \mathbb{E}(Y) = \frac{1}{n} \sum_{i} \mu_i](https://s0.wp.com/latex.php?latex=%5Cmu+%3D+%5Cmathbb%7BE%7D%28Y%29+%3D+%5Cfrac%7B1%7D%7Bn%7D+%5Csum_%7Bi%7D+%5Cmu_i&bg=ffffff&fg=36312d&s=0). Then the Chernoff-Hoeffding inequality gives an exponential upper bound on the probability that the value of ![Y](https://s0.wp.com/latex.php?latex=Y&bg=ffffff&fg=36312d&s=0) deviates from its mean. Specifically,

![\displaystyle \textup{P}(Y + a < \mu) \leq e^{-2na^2}](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Ctextup%7BP%7D%28Y+%2B+a+%3C+%5Cmu%29+%5Cleq+e%5E%7B-2na%5E2%7D&bg=ffffff&fg=36312d&s=0)

For us, the ![Y_i](https://s0.wp.com/latex.php?latex=Y_i&bg=ffffff&fg=36312d&s=0) will be the payoff variables for a single action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0) in the rounds for which we choose action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0). Then the variable ![Y](https://s0.wp.com/latex.php?latex=Y&bg=ffffff&fg=36312d&s=0) is just the empirical average payoff for action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0) over all the times we’ve tried it. Moreover, ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=36312d&s=0) is our one-sided upper bound (and as a lower bound, sometimes). We can then solve this equation for ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=36312d&s=0) to find an upper bound big enough to be confident that we’re within ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=36312d&s=0) of the true mean.

Indeed, if we call ![n_j](https://s0.wp.com/latex.php?latex=n_j&bg=ffffff&fg=36312d&s=0) the number of times we played action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0) thus far, then ![n = n_j](https://s0.wp.com/latex.php?latex=n+%3D+n_j&bg=ffffff&fg=36312d&s=0) in the equation above, and using ![a = a(j,T) = \sqrt{2 \log(T) / n_j}](https://s0.wp.com/latex.php?latex=a+%3D+a%28j%2CT%29+%3D+%5Csqrt%7B2+%5Clog%28T%29+%2F+n_j%7D&bg=ffffff&fg=36312d&s=0) we get that ![\textup{P}(Y > \mu + a) \leq T^{-4}](https://s0.wp.com/latex.php?latex=%5Ctextup%7BP%7D%28Y+%3E+%5Cmu+%2B+a%29+%5Cleq+T%5E%7B-4%7D&bg=ffffff&fg=36312d&s=0), which converges to zero very quickly as the number of rounds played grows. We’ll see this pop up again in the algorithm’s analysis below. But before that note two things. First, assuming we don’t play an action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0), its upper bound ![a](https://s0.wp.com/latex.php?latex=a&bg=ffffff&fg=36312d&s=0)grows in the number of rounds. This means that we never permanently rule out an action no matter how poorly it performs. If we get extremely unlucky with the optimal action, we will eventually be convinced to try it again. Second, the probability that our upper bound is wrong decreases in the number of rounds independently of how many times we’ve played the action. That is because our upper bound ![a(j, T)](https://s0.wp.com/latex.php?latex=a%28j%2C+T%29&bg=ffffff&fg=36312d&s=0) is getting bigger for actions we haven’t played; any round in which we play an action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0), it must be that ![a(j, T+1) = a(j,T)](https://s0.wp.com/latex.php?latex=a%28j%2C+T%2B1%29+%3D+a%28j%2CT%29&bg=ffffff&fg=36312d&s=0), although the empirical mean will likely change.

With these two facts in mind, we can formally state the algorithm and intuitively understand why it should work.

**UCB1:**
Play each of the ![K](https://s0.wp.com/latex.php?latex=K&bg=ffffff&fg=36312d&s=0) actions once, giving initial values for empirical mean payoffs ![\overline{x}_i](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D_i&bg=ffffff&fg=36312d&s=0) of each action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0).
For each round ![t = K, K+1, \dots](https://s0.wp.com/latex.php?latex=t+%3D+K%2C+K%2B1%2C+%5Cdots&bg=ffffff&fg=36312d&s=0):

Let ![n_j](https://s0.wp.com/latex.php?latex=n_j&bg=ffffff&fg=36312d&s=0) represent the number of times action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0) was played so far.
Play the action ![j](https://s0.wp.com/latex.php?latex=j&bg=ffffff&fg=36312d&s=0) maximizing ![\overline{x}_j + \sqrt{2 \log t / n_j}](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D_j+%2B+%5Csqrt%7B2+%5Clog+t+%2F+n_j%7D&bg=ffffff&fg=36312d&s=0).
Observe the reward ![X_{j,t}](https://s0.wp.com/latex.php?latex=X_%7Bj%2Ct%7D&bg=ffffff&fg=36312d&s=0) and update the empirical mean for the chosen action.

And that’s it. Note that we’re being super stateful here: the empirical means ![x_j](https://s0.wp.com/latex.php?latex=x_j&bg=ffffff&fg=36312d&s=0) change over time, and we’ll leave this update implicit throughout the rest of our discussion (sorry, functional programmers, but the notation is horrendous otherwise).

Before we implement and test this algorithm, let’s go ahead and prove that it achieves nearly optimal regret. The reader uninterested in mathematical details should skip the proof, but the discussion of the theorem itself is important. If one wants to use this algorithm in real life, one needs to understand the guarantees it provides in order to adequately quantify the risk involved in using it.

**Theorem:** Suppose that UCB1 is run on the bandit game with ![K](https://s0.wp.com/latex.php?latex=K&bg=ffffff&fg=36312d&s=0) actions, each of whose reward distribution ![X_{i,t}](https://s0.wp.com/latex.php?latex=X_%7Bi%2Ct%7D&bg=ffffff&fg=36312d&s=0) has values in [0,1]. Then its expected cumulative regret after ![T](https://s0.wp.com/latex.php?latex=T&bg=ffffff&fg=36312d&s=0) rounds is at most ![O(\sqrt{KT \log T})](https://s0.wp.com/latex.php?latex=O%28%5Csqrt%7BKT+%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0).

Actually, we’ll prove a more specific theorem. Let ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0) be the difference ![\mu^* - \mu_i](https://s0.wp.com/latex.php?latex=%5Cmu%5E%2A+-+%5Cmu_i&bg=ffffff&fg=36312d&s=0), where ![\mu^*](https://s0.wp.com/latex.php?latex=%5Cmu%5E%2A&bg=ffffff&fg=36312d&s=0) is the expected payoff of the best action, and let ![\Delta](https://s0.wp.com/latex.php?latex=%5CDelta&bg=ffffff&fg=36312d&s=0) be the minimal nonzero ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0). That is, ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0) represents how suboptimal an action is and ![\Delta](https://s0.wp.com/latex.php?latex=%5CDelta&bg=ffffff&fg=36312d&s=0) is the suboptimality of the second best action. These constants are called *problem-dependent constants.* The theorem we’ll actually prove is:

**Theorem:** Suppose UCB1 is run as above. Then its expected cumulative regret ![\mathbb{E}(R_{\textup{UCB1}}(T))](https://s0.wp.com/latex.php?latex=%5Cmathbb%7BE%7D%28R_%7B%5Ctextup%7BUCB1%7D%7D%28T%29%29&bg=ffffff&fg=36312d&s=0) is at most

![\displaystyle 8 \sum_{i : \mu_i < \mu^*} \frac{\log T}{\Delta_i} + \left ( 1 + \frac{\pi^2}{3} \right ) \left ( \sum_{j=1}^K \Delta_j \right )](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+8+%5Csum_%7Bi+%3A+%5Cmu_i+%3C+%5Cmu%5E%2A%7D+%5Cfrac%7B%5Clog+T%7D%7B%5CDelta_i%7D+%2B+%5Cleft+%28+1+%2B+%5Cfrac%7B%5Cpi%5E2%7D%7B3%7D+%5Cright+%29+%5Cleft+%28+%5Csum_%7Bj%3D1%7D%5EK+%5CDelta_j+%5Cright+%29&bg=ffffff&fg=36312d&s=0)

Okay, this looks like one nasty puppy, but it’s actually not that bad. The first term of the sum signifies that we expect to play any suboptimal machine about a logarithmic number of times, roughly scaled by how hard it is to distinguish from the optimal machine. That is, if ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0) is small we will require more tries to know that action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) is suboptimal, and hence we will incur more regret. The second term represents a small constant number (the ![1 + \pi^2 / 3](https://s0.wp.com/latex.php?latex=1+%2B+%5Cpi%5E2+%2F+3&bg=ffffff&fg=36312d&s=0) part) that caps the number of times we’ll play suboptimal machines in excess of the first term due to unlikely events occurring. So the first term is like our expected losses, and the second is our risk.

But note that this is a *worst-case bound* on the regret. We’re not saying we will achieve this much regret, or anywhere near it, but that UCB1 simply cannot do worse than this. Our hope is that in practice UCB1 performs much better.

Before we prove the theorem, let’s see how derive the ![O(\sqrt{KT \log T})](https://s0.wp.com/latex.php?latex=O%28%5Csqrt%7BKT+%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0) bound mentioned above. This will require familiarity with multivariable calculus, but such things must be endured like ripping off a band-aid. First consider the regret as a function ![R(\Delta_1, \dots, \Delta_K)](https://s0.wp.com/latex.php?latex=R%28%5CDelta_1%2C+%5Cdots%2C+%5CDelta_K%29&bg=ffffff&fg=36312d&s=0) (excluding of course ![\Delta^*](https://s0.wp.com/latex.php?latex=%5CDelta%5E%2A&bg=ffffff&fg=36312d&s=0)), and let’s look at the worst case bound by maximizing it. In particular, we’re just finding the problem with the parameters which screw our bound as badly as possible, The gradient of the regret function is given by

![\displaystyle \frac{\partial R}{\partial \Delta_i} = - \frac{8 \log T}{\Delta_i^2} + 1 + \frac{\pi^2}{3}](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Cfrac%7B%5Cpartial+R%7D%7B%5Cpartial+%5CDelta_i%7D+%3D+-+%5Cfrac%7B8+%5Clog+T%7D%7B%5CDelta_i%5E2%7D+%2B+1+%2B+%5Cfrac%7B%5Cpi%5E2%7D%7B3%7D&bg=ffffff&fg=36312d&s=0)

and it’s zero if and only if for each ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0), ![\Delta_i = \sqrt{\frac{8 \log T}{1 + \pi^2/3}} = O(\sqrt{\log T})](https://s0.wp.com/latex.php?latex=%5CDelta_i+%3D+%5Csqrt%7B%5Cfrac%7B8+%5Clog+T%7D%7B1+%2B+%5Cpi%5E2%2F3%7D%7D+%3D+O%28%5Csqrt%7B%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0). However this is a *minimum* of the regret bound (the Hessian is diagonal and all its eigenvalues are positive). Plugging in the ![\Delta_i = O(\sqrt{\log T})](https://s0.wp.com/latex.php?latex=%5CDelta_i+%3D+O%28%5Csqrt%7B%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0) (which are all the same) gives a total bound of ![O(K \sqrt{\log T})](https://s0.wp.com/latex.php?latex=O%28K+%5Csqrt%7B%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0). If we look at the only possible endpoint (the ![\Delta_i = 1](https://s0.wp.com/latex.php?latex=%5CDelta_i+%3D+1&bg=ffffff&fg=36312d&s=0)), then we get a local maximum of ![O(K \sqrt{\log T})](https://s0.wp.com/latex.php?latex=O%28K+%5Csqrt%7B%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0). But this isn’t the ![O(\sqrt{KT \log T})](https://s0.wp.com/latex.php?latex=O%28%5Csqrt%7BKT+%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0) we promised, what gives? Well, this upper bound grows arbitrarily large as the ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0) go to zero. But at the same time, if all the ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0) are small, then we shouldn’t be incurring much regret because we’ll be picking actions that are close to optimal!

Indeed, if we assume for simplicity that all the ![\Delta_i = \Delta](https://s0.wp.com/latex.php?latex=%5CDelta_i+%3D+%5CDelta&bg=ffffff&fg=36312d&s=0) are the same, then another trivial regret bound is ![\Delta T](https://s0.wp.com/latex.php?latex=%5CDelta+T&bg=ffffff&fg=36312d&s=0) (why?). The true regret is hence the minimum of this regret bound and the UCB1 regret bound: as the UCB1 bound degrades we will eventually switch to the simpler bound. That will be a non-differentiable switch (and hence a critical point) and it occurs at ![\Delta = O(\sqrt{(K \log T) / T})](https://s0.wp.com/latex.php?latex=%5CDelta+%3D+O%28%5Csqrt%7B%28K+%5Clog+T%29+%2F+T%7D%29&bg=ffffff&fg=36312d&s=0). Hence the regret bound at the switch is ![\Delta T = O(\sqrt{KT \log T})](https://s0.wp.com/latex.php?latex=%5CDelta+T+%3D+O%28%5Csqrt%7BKT+%5Clog+T%7D%29&bg=ffffff&fg=36312d&s=0), as desired.

## Proving the Worst-Case Regret Bound

*Proof.* The proof works by finding a bound on ![P_i(T)](https://s0.wp.com/latex.php?latex=P_i%28T%29&bg=ffffff&fg=36312d&s=0), the expected number of times UCB chooses an action up to round ![T](https://s0.wp.com/latex.php?latex=T&bg=ffffff&fg=36312d&s=0). Using the ![\Delta](https://s0.wp.com/latex.php?latex=%5CDelta&bg=ffffff&fg=36312d&s=0) notation, the regret is then just ![\sum_i \Delta_i \mathbb{E}(P_i(T))](https://s0.wp.com/latex.php?latex=%5Csum_i+%5CDelta_i+%5Cmathbb%7BE%7D%28P_i%28T%29%29&bg=ffffff&fg=36312d&s=0), and bounding the ![P_i](https://s0.wp.com/latex.php?latex=P_i&bg=ffffff&fg=36312d&s=0)‘s will bound the regret.

Recall the notation for our upper bound ![a(j, T) = \sqrt{2 \log T / P_j(T)}](https://s0.wp.com/latex.php?latex=a%28j%2C+T%29+%3D+%5Csqrt%7B2+%5Clog+T+%2F+P_j%28T%29%7D&bg=ffffff&fg=36312d&s=0) and let’s loosen it a bit to ![a(y, T) = \sqrt{2 \log T / y}](https://s0.wp.com/latex.php?latex=a%28y%2C+T%29+%3D+%5Csqrt%7B2+%5Clog+T+%2F+y%7D&bg=ffffff&fg=36312d&s=0) so that we’re allowed to “pretend” a action has been played ![y](https://s0.wp.com/latex.php?latex=y&bg=ffffff&fg=36312d&s=0) times. Recall further that the random variable ![I_t](https://s0.wp.com/latex.php?latex=I_t&bg=ffffff&fg=36312d&s=0) has as its value the index of the machine chosen. We denote by ![\chi(E)](https://s0.wp.com/latex.php?latex=%5Cchi%28E%29&bg=ffffff&fg=36312d&s=0) the indicator random variable for the event ![E](https://s0.wp.com/latex.php?latex=E&bg=ffffff&fg=36312d&s=0). And remember that we use an asterisk to denote a quantity associated with the optimal action (e.g., ![\overline{x}^*](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D%5E%2A&bg=ffffff&fg=36312d&s=0) is the empirical mean of the optimal action).

Indeed for any action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0), the only way we know how to write down ![P_i(T)](https://s0.wp.com/latex.php?latex=P_i%28T%29&bg=ffffff&fg=36312d&s=0) is as

![\displaystyle P_i(T) = 1 + \sum_{t=K}^T \chi(I_t = i)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+P_i%28T%29+%3D+1+%2B+%5Csum_%7Bt%3DK%7D%5ET+%5Cchi%28I_t+%3D+i%29&bg=ffffff&fg=36312d&s=0)

The 1 is from the initialization where we play each action once, and the sum is the trivial thing where just count the number of rounds in which we pick action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0). Now we’re just going to pull some number ![m-1](https://s0.wp.com/latex.php?latex=m-1&bg=ffffff&fg=36312d&s=0) of plays out of that summation, keep it variable, and try to optimize over it. Since we might play the action fewer than ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0)times overall, this requires an inequality.

![P_i(T) \leq m + \sum_{t=K}^T \chi(I_t = i \textup{ and } P_i(t-1) \geq m)](https://s0.wp.com/latex.php?latex=P_i%28T%29+%5Cleq+m+%2B+%5Csum_%7Bt%3DK%7D%5ET+%5Cchi%28I_t+%3D+i+%5Ctextup%7B+and+%7D+P_i%28t-1%29+%5Cgeq+m%29&bg=ffffff&fg=36312d&s=0)

These indicator functions should be read as sentences: we’re just saying that we’re picking action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) in round ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) and we’ve already played ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) at least ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0) times. Now we’re going to focus on the inside of the summation, and come up with an event that happens at least as frequently as this one to get an upper bound. Specifically, saying that we’ve picked action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) in round ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) means that the upper bound for action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) exceeds the upper bound for every other action. In particular, this means its upper bound exceeds the upper bound of the best action (and ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) might coincide with the best action, but that’s fine). In notation this event is

![\displaystyle \overline{x}_i + a(P_i(t), t-1) \geq \overline{x}^* + a(P^*(T), t-1)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Coverline%7Bx%7D_i+%2B+a%28P_i%28t%29%2C+t-1%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A+%2B+a%28P%5E%2A%28T%29%2C+t-1%29&bg=ffffff&fg=36312d&s=0)

Denote the upper bound ![\overline{x}_i + a(i,t)](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D_i+%2B+a%28i%2Ct%29&bg=ffffff&fg=36312d&s=0) for action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) in round ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) by ![U_i(t)](https://s0.wp.com/latex.php?latex=U_i%28t%29&bg=ffffff&fg=36312d&s=0). Since this event must occur every time we pick action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) (though not necessarily vice versa), we have

![\displaystyle P_i(T) \leq m + \sum_{t=K}^T \chi(U_i(t-1) \geq U^*(t-1) \textup{ and } P_i(t-1) \geq m)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+P_i%28T%29+%5Cleq+m+%2B+%5Csum_%7Bt%3DK%7D%5ET+%5Cchi%28U_i%28t-1%29+%5Cgeq+U%5E%2A%28t-1%29+%5Ctextup%7B+and+%7D+P_i%28t-1%29+%5Cgeq+m%29&bg=ffffff&fg=36312d&s=0)

We’ll do this process again but with a slightly more complicated event. If the upper bound of action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) exceeds that of the optimal machine, it is also the case that the maximum upper bound for action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) we’ve seen after the first ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0) trials exceeds the minimum upper bound we’ve seen on the optimal machine (ever). But on round ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) we don’t know how many times we’ve played the optimal machine, nor do we even know how many times we’ve played machine ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) (except that it’s more than ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0)). So we try all possibilities and look at minima and maxima. This is a pretty crude approximation, but it will allow us to write things in a nicer form.

Denote by ![\overline{x}_{i,s}](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D_%7Bi%2Cs%7D&bg=ffffff&fg=36312d&s=0) the random variable for the empirical mean after playing action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) a total of ![s](https://s0.wp.com/latex.php?latex=s&bg=ffffff&fg=36312d&s=0) times, and ![\overline{x}^*_s](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D%5E%2A_s&bg=ffffff&fg=36312d&s=0) the corresponding quantity for the optimal machine. Realizing everything in notation, the above argument proves that

![\displaystyle P_i(T) \leq m + \sum_{t=K}^T \chi \left ( \max_{m \leq s < t} \overline{x}_{i,s} + a(s, t-1) \geq \min_{0 < s' < t} \overline{x}^*_{s'} + a(s', t-1) \right )](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+P_i%28T%29+%5Cleq+m+%2B+%5Csum_%7Bt%3DK%7D%5ET+%5Cchi+%5Cleft+%28+%5Cmax_%7Bm+%5Cleq+s+%3C+t%7D+%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t-1%29+%5Cgeq+%5Cmin_%7B0+%3C+s%27+%3C+t%7D+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t-1%29+%5Cright+%29&bg=ffffff&fg=36312d&s=0)

Indeed, at each ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) for which the max is greater than the min, there will be at least one pair ![s,s'](https://s0.wp.com/latex.php?latex=s%2Cs%27&bg=ffffff&fg=36312d&s=0) for which the values of the quantities inside the max/min will satisfy the inequality. And so, even worse, we can just count the number of pairs ![s, s'](https://s0.wp.com/latex.php?latex=s%2C+s%27&bg=ffffff&fg=36312d&s=0) for which it happens. That is, we can expand the event above into the double sum which is at least as large:

![\displaystyle P_i(T) \leq m + \sum_{t=K}^T \sum_{s = m}^{t-1} \sum_{s' = 1}^{t-1} \chi \left ( \overline{x}_{i,s} + a(s, t-1) \geq \overline{x}^*_{s'} + a(s', t-1) \right )](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+P_i%28T%29+%5Cleq+m+%2B+%5Csum_%7Bt%3DK%7D%5ET+%5Csum_%7Bs+%3D+m%7D%5E%7Bt-1%7D+%5Csum_%7Bs%27+%3D+1%7D%5E%7Bt-1%7D+%5Cchi+%5Cleft+%28+%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t-1%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t-1%29+%5Cright+%29&bg=ffffff&fg=36312d&s=0)

We can make one other odd inequality by increasing the sum to go from ![t=1](https://s0.wp.com/latex.php?latex=t%3D1&bg=ffffff&fg=36312d&s=0) to ![\infty](https://s0.wp.com/latex.php?latex=%5Cinfty&bg=ffffff&fg=36312d&s=0). This will become clear later, but it means we can replace ![t-1](https://s0.wp.com/latex.php?latex=t-1&bg=ffffff&fg=36312d&s=0) with ![t](https://s0.wp.com/latex.php?latex=t&bg=ffffff&fg=36312d&s=0) and thus have

![\displaystyle P_i(T) \leq m + \sum_{t=1}^\infty \sum_{s = m}^{t-1} \sum_{s' = 1}^{t-1} \chi \left ( \overline{x}_{i,s} + a(s, t) \geq \overline{x}^*_{s'} + a(s', t) \right )](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+P_i%28T%29+%5Cleq+m+%2B+%5Csum_%7Bt%3D1%7D%5E%5Cinfty+%5Csum_%7Bs+%3D+m%7D%5E%7Bt-1%7D+%5Csum_%7Bs%27+%3D+1%7D%5E%7Bt-1%7D+%5Cchi+%5Cleft+%28+%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t%29+%5Cright+%29&bg=ffffff&fg=36312d&s=0)

Now that we’ve slogged through this mess of inequalities, we can actually get to the heart of the argument. Suppose that this event actually happens, that ![\overline{x}_{i,s} + a(s, t) \geq \overline{x}^*_{s'} + a(s', t)](https://s0.wp.com/latex.php?latex=%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t%29&bg=ffffff&fg=36312d&s=0). Then what can we say? Well, consider the following three events:

(1) ![\displaystyle \overline{x}^*_{s'} \leq \mu^* - a(s', t)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%5Cleq+%5Cmu%5E%2A+-+a%28s%27%2C+t%29&bg=ffffff&fg=36312d&s=0)
(2) ![\displaystyle \overline{x}_{i,s} \geq \mu_i + a(s, t)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%5Cgeq+%5Cmu_i+%2B+a%28s%2C+t%29&bg=ffffff&fg=36312d&s=0)
(3) ![\displaystyle \mu^* < \mu_i + 2a(s, t)](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Cmu%5E%2A+%3C+%5Cmu_i+%2B+2a%28s%2C+t%29&bg=ffffff&fg=36312d&s=0)

In words, (1) is the event that the empirical mean of the optimal action is less than the lower confidence bound. By our Chernoff bound argument earlier, this happens with probability ![t^{-4}](https://s0.wp.com/latex.php?latex=t%5E%7B-4%7D&bg=ffffff&fg=36312d&s=0). Likewise, (2) is the event that the empirical mean payoff of action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) is larger than the upper confidence bound, which also occurs with probability ![t^{-4}](https://s0.wp.com/latex.php?latex=t%5E%7B-4%7D&bg=ffffff&fg=36312d&s=0). We will see momentarily that (3) is impossible for a well-chosen ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0) (which is why we left it variable), but in any case the claim is that one of these three events must occur. For if they are all false, we have

![\displaystyle \begin{matrix} \overline{x}_{i,s} + a(s, t) \geq \overline{x}^*_{s'} + a(s', t) & > & \mu^* - a(s',t) + a(s',t) = \mu^* \\ \textup{assumed} & (1) \textup{ is false} & \\ \end{matrix}](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Cbegin%7Bmatrix%7D+%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t%29+%26+%3E+%26+%5Cmu%5E%2A+-+a%28s%27%2Ct%29+%2B+a%28s%27%2Ct%29+%3D+%5Cmu%5E%2A+%5C%5C+%5Ctextup%7Bassumed%7D+%26+%281%29+%5Ctextup%7B+is+false%7D+%26+%5C%5C+%5Cend%7Bmatrix%7D&bg=ffffff&fg=36312d&s=0)

and

![\begin{matrix} \mu_i + 2a(s,t) & > & \overline{x}_{i,s} + a(s, t) \geq \overline{x}^*_{s'} + a(s', t) \\ & (2) \textup{ is false} & \textup{assumed} \\ \end{matrix}](https://s0.wp.com/latex.php?latex=%5Cbegin%7Bmatrix%7D+%5Cmu_i+%2B+2a%28s%2Ct%29+%26+%3E+%26+%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t%29+%5C%5C+%26+%282%29+%5Ctextup%7B+is+false%7D+%26+%5Ctextup%7Bassumed%7D+%5C%5C+%5Cend%7Bmatrix%7D&bg=ffffff&fg=36312d&s=0)

But putting these two inequalities together gives us precisely that (3) is true:

![\mu^* < \mu_i + 2a(s,t)](https://s0.wp.com/latex.php?latex=%5Cmu%5E%2A+%3C+%5Cmu_i+%2B+2a%28s%2Ct%29&bg=ffffff&fg=36312d&s=0)

This proves the claim.

By the [union bound](http://en.wikipedia.org/wiki/Boole's_inequality), the probability that at least one of these events happens is ![2t^{-4}](https://s0.wp.com/latex.php?latex=2t%5E%7B-4%7D&bg=ffffff&fg=36312d&s=0)plus whatever the probability of (3) being true is. But as we said, we’ll pick ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0) to make (3) always false. Indeed ![m](https://s0.wp.com/latex.php?latex=m&bg=ffffff&fg=36312d&s=0) depends on which action ![i](https://s0.wp.com/latex.php?latex=i&bg=ffffff&fg=36312d&s=0) is being played, and if ![s \geq m > 8 \log T / \Delta_i^2](https://s0.wp.com/latex.php?latex=s+%5Cgeq+m+%3E+8+%5Clog+T+%2F+%5CDelta_i%5E2&bg=ffffff&fg=36312d&s=0) then ![2a(s,t) \leq \Delta_i](https://s0.wp.com/latex.php?latex=2a%28s%2Ct%29+%5Cleq+%5CDelta_i&bg=ffffff&fg=36312d&s=0), and by the definition of ![\Delta_i](https://s0.wp.com/latex.php?latex=%5CDelta_i&bg=ffffff&fg=36312d&s=0) we have

![\mu^* - \mu_i - 2a(s,t) \geq \mu^* - \mu_i - \Delta_i = 0](https://s0.wp.com/latex.php?latex=%5Cmu%5E%2A+-+%5Cmu_i+-+2a%28s%2Ct%29+%5Cgeq+%5Cmu%5E%2A+-+%5Cmu_i+-+%5CDelta_i+%3D+0&bg=ffffff&fg=36312d&s=0).

Now we can finally piece everything together. The expected value of an event is just its probability of occurring, and so

![\displaystyle \begin{aligned} \mathbb{E}(P_i(T)) & \leq m + \sum_{t=1}^\infty \sum_{s=m}^t \sum_{s' = 1}^t \textup{P}(\overline{x}_{i,s} + a(s, t) \geq \overline{x}^*_{s'} + a(s', t)) \\ & \leq \left \lceil \frac{8 \log T}{\Delta_i^2} \right \rceil + \sum_{t=1}^\infty \sum_{s=m}^t \sum_{s' = 1}^t 2t^{-4} \\ & \leq \frac{8 \log T}{\Delta_i^2} + 1 + \sum_{t=1}^\infty \sum_{s=1}^t \sum_{s' = 1}^t 2t^{-4} \\ & = \frac{8 \log T}{\Delta_i^2} + 1 + 2 \sum_{t=1}^\infty t^{-2} \\ & = \frac{8 \log T}{\Delta_i^2} + 1 + \frac{\pi^2}{3} \\ \end{aligned}](https://s0.wp.com/latex.php?latex=%5Cdisplaystyle+%5Cbegin%7Baligned%7D+%5Cmathbb%7BE%7D%28P_i%28T%29%29+%26+%5Cleq+m+%2B+%5Csum_%7Bt%3D1%7D%5E%5Cinfty+%5Csum_%7Bs%3Dm%7D%5Et+%5Csum_%7Bs%27+%3D+1%7D%5Et+%5Ctextup%7BP%7D%28%5Coverline%7Bx%7D_%7Bi%2Cs%7D+%2B+a%28s%2C+t%29+%5Cgeq+%5Coverline%7Bx%7D%5E%2A_%7Bs%27%7D+%2B+a%28s%27%2C+t%29%29+%5C%5C+%26+%5Cleq+%5Cleft+%5Clceil+%5Cfrac%7B8+%5Clog+T%7D%7B%5CDelta_i%5E2%7D+%5Cright+%5Crceil+%2B+%5Csum_%7Bt%3D1%7D%5E%5Cinfty+%5Csum_%7Bs%3Dm%7D%5Et+%5Csum_%7Bs%27+%3D+1%7D%5Et+2t%5E%7B-4%7D+%5C%5C+%26+%5Cleq+%5Cfrac%7B8+%5Clog+T%7D%7B%5CDelta_i%5E2%7D+%2B+1+%2B+%5Csum_%7Bt%3D1%7D%5E%5Cinfty+%5Csum_%7Bs%3D1%7D%5Et+%5Csum_%7Bs%27+%3D+1%7D%5Et+2t%5E%7B-4%7D+%5C%5C+%26+%3D+%5Cfrac%7B8+%5Clog+T%7D%7B%5CDelta_i%5E2%7D+%2B+1+%2B+2+%5Csum_%7Bt%3D1%7D%5E%5Cinfty+t%5E%7B-2%7D+%5C%5C+%26+%3D+%5Cfrac%7B8+%5Clog+T%7D%7B%5CDelta_i%5E2%7D+%2B+1+%2B+%5Cfrac%7B%5Cpi%5E2%7D%7B3%7D+%5C%5C+%5Cend%7Baligned%7D&bg=ffffff&fg=36312d&s=0)

The second line is the Chernoff bound we argued above, the third and fourth lines are relatively obvious algebraic manipulations, and the last equality uses the classic solution to [Basel’s problem](http://en.wikipedia.org/wiki/Basel_problem). Plugging this upper bound in to the regret formula we gave in the first paragraph of the proof establishes the bound and proves the theorem.

## Implementation and an Experiment

The algorithm is about as simple to write in code as it is in pseudocode. The confidence bound is trivial to implement (though note we index from zero):

```
`def` `upperBound(step, numPlays):``   ``return` `math.sqrt(``2` `*` `math.log(step ``+` `1``) ``/` `numPlays)`
```

And the full algorithm is quite short as well. We define a function ub1, which accepts as input the number of actions and a function reward which accepts as input the index of the action and the time step, and draws from the appropriate reward distribution. Then implementing ub1 is simply a matter of keeping track of empirical averages and an argmax. We implement the function as a Python generator, so one can observe the steps of the algorithm and keep track of the confidence bounds and the cumulative regret.
